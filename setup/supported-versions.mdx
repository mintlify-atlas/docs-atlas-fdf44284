---
title: "Supported Versions"
description: "The Minecraft: Bedrock Edition and Java Edition versions that Geyser currently supports."
---

Geyser acts as a protocol translator between Bedrock and Java editions. Because each edition uses a different protocol, Geyser must explicitly support each version.

## Current supported versions

| Edition | Supported versions |
|---|---|
| Bedrock | 1.21.130, 1.21.131, 1.21.132, 26.0, 26.1, 26.2, 26.3, 26.10 |
| Java | 1.21.1 |

<Note>
  Geyser supports exactly **one** Java Edition version at a time. Your Java server must be running the version listed above. Geyser does not support older or newer Java protocol versions.
</Note>

## Bedrock version details

Bedrock Edition has two versioning schemes in active use. Versions in the `1.21.x` range use the traditional scheme, while `26.x` versions use the newer scheme introduced alongside Minecraft Preview updates.

| Protocol version | Bedrock game versions |
|---|---|
| v898 | 1.21.130, 1.21.131, 1.21.132 |
| v924 | 26.0, 26.1, 26.2, 26.3 |
| v944 | 26.10 |

All listed Bedrock versions can connect to a single Geyser instance simultaneously.

## Bedrock auto-updates

Minecraft: Bedrock Edition updates automatically on most platforms (consoles, mobile, Windows). This means players may be on a newer Bedrock version than Geyser currently supports.

<Warning>
  If a player's Bedrock client is newer than Geyser supports, they will see a "Outdated server" or version mismatch error and will not be able to connect until Geyser is updated.
</Warning>

Geyser can alert server operators when a newer Bedrock version is detected. This is controlled by the `notifyOnNewBedrockUpdate` option in `config.yml`:

```yaml
# Whether to alert the console and operators that a new Geyser version is available
# that supports a Bedrock version that this Geyser version does not support.
# Recommended: keep this enabled, as many Bedrock platforms auto-update.
notify-on-new-bedrock-update: true
```

When this is enabled, server operators and console will receive a message when a Bedrock client connects with an unsupported protocol version, prompting you to update Geyser.

<Tip>
  Watch the [GeyserMC Discord](https://discord.gg/geysermc) `#announcements` channel for release notices whenever a new Bedrock update requires a Geyser update.
</Tip>

## Older Java versions

If your Java server is running a version older than the currently supported one, Geyser cannot connect to it directly. Options include:

- **Update your Java server** to the supported version (recommended).
- **Use ViaVersion** on your Java server alongside Geyser to support multiple Java protocol versions. See the [older versions guide](https://geysermc.org/wiki/geyser/supported-versions/) for details.

## Checking versions at runtime

You can see the exact protocol versions Geyser has loaded by running the following command in your server console:

```
geyser version
```

The output includes the Geyser build version, the supported Java protocol version, and all supported Bedrock protocol versions.
