---
title: "Custom Items"
description: "Register custom Bedrock items that appear when Java players hold specific items or meet specific predicates."
---

Geyser can display custom Bedrock items to Bedrock players by mapping Java item stacks to a Bedrock item definition. The API provides two approaches:

- **v2 API (`CustomItemDefinition`)** — the current, recommended API based on Java item model definitions and predicates
- **v1 API (`CustomItemData`)** — deprecated legacy API, still supported for backwards compatibility

Custom items must be registered during `GeyserDefineCustomItemsEvent`, which fires once at startup.

<Note>
  Custom item registration requires the `enable-custom-content` setting to be enabled in the Geyser config (it is enabled by default). This event will not be called if `enable-custom-content` is `false`.
</Note>

---

## Registering items: event hook

```java
import org.geysermc.event.subscribe.Subscribe;
import org.geysermc.geyser.api.event.lifecycle.GeyserDefineCustomItemsEvent;

public class MyExtension implements Extension {

    @Subscribe
    public void onDefineCustomItems(GeyserDefineCustomItemsEvent event) {
        // Register your items here
    }
}
```

## CustomItemDefinition (v2 — recommended)

`CustomItemDefinition` maps a Bedrock item identifier to a Java item model. A definition is matched when:

1. The Java item matches the vanilla item the definition is registered for
2. The item's active model matches `definition.model()`
3. All predicates (if any) pass

### Basic usage

```java
import org.geysermc.geyser.api.item.custom.v2.CustomItemDefinition;
import org.geysermc.geyser.api.util.Identifier;

// Register a custom item for the "minecraft:diamond_sword" Java item
// when its model is "my_pack:cool_sword"
Identifier bedrockId = new Identifier("my_pack", "cool_sword");
Identifier model    = new Identifier("my_pack", "cool_sword");

CustomItemDefinition definition = CustomItemDefinition
    .builder(bedrockId, model)
    .displayName("Cool Sword")
    .build();

event.register(new Identifier("minecraft", "diamond_sword"), definition);
```

### Adding predicates

Predicates allow fine-grained control over when a definition is selected. Multiple predicates are evaluated using AND logic by default.

```java
import org.geysermc.geyser.api.predicate.item.ItemConditionPredicate;
import org.geysermc.geyser.api.predicate.item.ItemRangeDispatchPredicate;
import org.geysermc.geyser.api.predicate.PredicateStrategy;

CustomItemDefinition def = CustomItemDefinition
    .builder(bedrockId, model)
    .predicate(/* some predicate */)
    .predicateStrategy(PredicateStrategy.AND) // default
    .priority(10) // higher priority evaluated first
    .build();
```

### Setting Bedrock options

Bedrock-specific item properties (icon, offhand, etc.) are set via `CustomItemBedrockOptions`:

```java
import org.geysermc.geyser.api.item.custom.v2.CustomItemBedrockOptions;

CustomItemDefinition def = CustomItemDefinition
    .builder(bedrockId, model)
    .bedrockOptions(
        CustomItemBedrockOptions.builder()
            // builder methods set icon, allow offhand, etc.
    )
    .build();
```

### Setting data components

Components describe Java item behaviors that the custom item must match on the server. Only specify components that are **always** present server-side:

```java
import org.geysermc.geyser.api.item.custom.v2.component.java.JavaItemDataComponents;

CustomItemDefinition def = CustomItemDefinition
    .builder(bedrockId, model)
    .component(JavaItemDataComponents.MAX_STACK_SIZE, 1)
    .component(JavaItemDataComponents.MAX_DAMAGE, 500)
    .build();
```

Supported components include `CONSUMABLE`, `EQUIPPABLE`, `FOOD`, `MAX_DAMAGE`, `MAX_STACK_SIZE`, `USE_COOLDOWN`, `ENCHANTABLE`, `TOOL`, `REPAIRABLE`, `ENCHANTMENT_GLINT_OVERRIDE`, `ATTACK_RANGE`, `PIERCING_WEAPON`, `USE_EFFECTS`, and `KINETIC_WEAPON`.

To remove a vanilla component from the custom item (must match server-side behavior):

```java
CustomItemDefinition def = CustomItemDefinition
    .builder(bedrockId, model)
    .removeComponent(JavaItemDataComponents.MAX_DAMAGE)
    .build();
```

### Priority and ordering

For a given Java item model, Geyser evaluates definitions in this order:

1. Higher `priority()` values first
2. Definitions with a similar `ItemRangeDispatchPredicate` — higher threshold first
3. Most predicates to fewest

## CustomItemData (v1 — deprecated)

<Warning>
  `CustomItemData` and `NonVanillaCustomItemData` are deprecated. Use `CustomItemDefinition` for new code.
</Warning>

The v1 API uses `CustomItemOptions` (NBT/damage predicates) to decide when to display the custom item. It is still supported via automatic conversion to `CustomItemDefinition` internally.

### Building a custom item (v1)

```java
import org.geysermc.geyser.api.item.custom.CustomItemData;
import org.geysermc.geyser.api.item.custom.CustomItemOptions;

CustomItemData item = CustomItemData.builder()
    .name("my_custom_sword")
    .displayName("My Custom Sword")
    .icon("my_custom_sword")
    .allowOffhand(true)
    .displayHandheld(true)
    .customItemOptions(myOptions) // CustomItemOptions with predicates
    .build();

event.register("minecraft:diamond_sword", item);
```

## Non-vanilla custom items

Non-vanilla custom items have **no Java counterpart** — they are typically added by mods. Use `NonVanillaCustomItemDefinition` (v2) or the deprecated `NonVanillaCustomItemData` (v1).

<Warning>
  `NonVanillaCustomItemData` is deprecated. Prefer `NonVanillaCustomItemDefinition` for new code.
</Warning>

```java
import org.geysermc.geyser.api.item.custom.NonVanillaCustomItemData;

NonVanillaCustomItemData item = NonVanillaCustomItemData.builder()
    .name("my_mod_item")
    .identifier("mymod:my_item")      // Java item identifier from the mod
    .displayName("My Mod Item")
    .icon("my_mod_item")
    .stackSize(64)
    .maxDamage(0)
    .attackDamage(5)
    .toolType("sword")
    .edible(false)
    .isHat(false)
    .isFoil(false)
    .isChargeable(false)
    .displayHandheld(true)
    .build();

event.register(item);
```

## Complete v2 registration example

```java
@Subscribe
public void onDefineCustomItems(GeyserDefineCustomItemsEvent event) {
    // A custom sword skin triggered by item model "my_pack:custom_sword"
    Identifier bedrockId = new Identifier("my_pack", "custom_sword");
    Identifier model     = new Identifier("my_pack", "custom_sword");

    CustomItemDefinition definition = CustomItemDefinition
        .builder(bedrockId, model)
        .displayName("Custom Sword")
        .priority(5)
        .build();

    event.register(new Identifier("minecraft", "diamond_sword"), definition);
}
```
