---
title: "Custom Blocks"
description: "Register custom Bedrock blocks with full component support, permutations, and non-vanilla block overrides."
---

Geyser can show custom Bedrock blocks to Bedrock players while mapping them to Java block states. Custom blocks must be registered once at startup during `GeyserDefineCustomBlocksEvent`.

<Note>
  Custom block registration requires the `enable-custom-content` setting to be enabled in the Geyser config (enabled by default). This event will not be called if `enable-custom-content` is `false`.
</Note>

---

## Registering blocks: event hook

```java
import org.geysermc.event.subscribe.Subscribe;
import org.geysermc.geyser.api.event.lifecycle.GeyserDefineCustomBlocksEvent;

public class MyExtension implements Extension {

    @Subscribe
    public void onDefineCustomBlocks(GeyserDefineCustomBlocksEvent event) {
        // Register custom blocks here
    }
}
```

---

## CustomBlockData

`CustomBlockData` (in `org.geysermc.geyser.api.block.custom`) describes a custom Bedrock block. Build it with `CustomBlockData.builder()`.

### Minimal example

```java
import org.geysermc.geyser.api.block.custom.CustomBlockData;
import org.geysermc.geyser.api.block.custom.component.CustomBlockComponents;

CustomBlockComponents components = CustomBlockComponents.builder()
    .displayName("My Custom Block")
    .destructibleByMining(2.0f)
    .friction(0.6f)
    .lightEmission(7)
    .build();

CustomBlockData block = CustomBlockData.builder()
    .name("my_custom_block")
    .includedInCreativeInventory(true)
    .components(components)
    .build();

event.register(block);
```

### Overriding a Java block state

```java
// Override a specific Java block state with your custom block's state
event.registerOverride("minecraft:stone[variant=granite]", block.defaultBlockState());

// Override based on a Java item identifier
event.registerItemOverride("minecraft:stone", block);
```

Java block state identifiers are listed [in the Geyser mappings](https://raw.githubusercontent.com/GeyserMC/mappings/master/blocks.json).

## CustomBlockComponents

`CustomBlockComponents` defines the Bedrock-side behavior of the block. Build it with `CustomBlockComponents.builder()`.

### Builder methods

```java
CustomBlockComponents components = CustomBlockComponents.builder()
    .displayName("My Block")                          // minecraft:display_name
    .geometry(geometryComponent)                      // minecraft:geometry
    .selectionBox(selectionBox)                       // minecraft:selection_box
    .collisionBoxes(collisionBox1, collisionBox2)     // minecraft:collision_box (up to 16)
    .materialInstance("*", materialInstance)          // minecraft:material_instances
    .destructibleByMining(3.0f)                       // minecraft:destructible_by_mining
    .friction(0.4f)                                   // minecraft:friction
    .lightEmission(15)                                // minecraft:light_emission
    .lightDampening(0)                                // minecraft:light_dampening
    .transformation(transformationComponent)          // minecraft:transformation
    .placeAir(false)                                  // place air instead of the block
    .tags(Set.of("my_tag", "another_tag"))            // tag:some_tag
    .build();
```

## Block permutations

Permutations let a single block have different components per block state. Define custom properties first, then attach permutations:

```java
import org.geysermc.geyser.api.block.custom.CustomBlockPermutation;

// A block with a boolean property "lit" that changes light emission
CustomBlockData block = CustomBlockData.builder()
    .name("my_lamp")
    .booleanProperty("my_pack:lit")
    .permutations(List.of(
        new CustomBlockPermutation(
            litComponents,    // CustomBlockComponents for the lit state
            "q.block_property('my_pack:lit') == true"
        )
    ))
    .components(defaultComponents) // components used when no permutation matches
    .build();
```

### Adding properties

```java
builder.booleanProperty("my_pack:is_powered")
builder.intProperty("my_pack:stage", List.of(0, 1, 2, 3))
builder.stringProperty("my_pack:variant", List.of("oak", "spruce", "birch"))
```

### Accessing block states

```java
CustomBlockState defaultState = block.defaultBlockState();

// Build a specific state
CustomBlockState litState = block.blockStateBuilder()
    // set property values on the builder
    .build();
```

## Non-vanilla custom blocks

`NonVanillaCustomBlockData` describes a completely custom block with **no Java counterpart**. It extends `CustomBlockData` and adds a `namespace()` field:

```java
import org.geysermc.geyser.api.block.custom.NonVanillaCustomBlockData;
import org.geysermc.geyser.api.block.custom.nonvanilla.JavaBlockState;

NonVanillaCustomBlockData block = NonVanillaCustomBlockData.builder()
    .namespace("my_mod")
    .name("my_custom_ore")
    .includedInCreativeInventory(true)
    .components(components)
    .build();

// You must also provide a JavaBlockState for non-vanilla blocks
JavaBlockState javaState = /* create JavaBlockState */;
event.registerOverride(javaState, block.defaultBlockState());
```

<ResponseField name="namespace()" type="String">
  The namespace used for the Bedrock block identifier (e.g. `my_mod`).
</ResponseField>

## Complete example

```java
@Subscribe
public void onDefineCustomBlocks(GeyserDefineCustomBlocksEvent event) {
    // Build components
    CustomBlockComponents components = CustomBlockComponents.builder()
        .displayName("Glowing Ore")
        .destructibleByMining(4.5f)
        .friction(0.6f)
        .lightEmission(8)
        .tags(Set.of("ore"))
        .build();

    // Build the block
    CustomBlockData glowingOre = CustomBlockData.builder()
        .name("glowing_ore")
        .includedInCreativeInventory(true)
        .components(components)
        .build();

    // Register the block
    event.register(glowingOre);

    // Override a Java block state with it
    event.registerOverride(
        "minecraft:iron_ore",
        glowingOre.defaultBlockState()
    );
}
```
