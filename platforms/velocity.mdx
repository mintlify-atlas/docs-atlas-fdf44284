---
title: "Velocity Setup"
description: "Install Geyser on a Velocity proxy to bridge Bedrock players across your entire network."
---

When you run a Velocity proxy network, install Geyser on the **proxy**, not on individual backend servers. This allows Bedrock players to connect to any server on your network through a single entry point.

## Prerequisites

- Java 17 or later
- Velocity 3.x or later
- Backend servers running Spigot 1.13.2 or later (Paper recommended)

## Installation

<Steps>
  <Step title="Download Geyser">
    Go to [https://geysermc.org/download](https://geysermc.org/download) and download **Geyser-Velocity**.
  </Step>

  <Step title="Place the JAR in your proxy plugins folder">
    Copy the downloaded `Geyser-Velocity.jar` into the `plugins/` folder of your **Velocity proxy**, not a backend server.

    ```
    proxy/
    └── plugins/
        └── Geyser-Velocity.jar
    ```
  </Step>

  <Step title="Configure player info forwarding in velocity.toml">
    Geyser requires that Velocity forwards player information to backend servers. Open `velocity.toml` and set `player-info-forwarding-mode` to either `"bungeecord"` (legacy) or `"modern"` (recommended):

    ```toml velocity.toml
    # Choose one of: "none", "legacy", "bungeecord", "modern"
    player-info-forwarding-mode = "modern"
    ```

    <Warning>
      If `player-info-forwarding-mode` is set to `"none"`, player data such as skins and UUIDs will not be forwarded correctly to backend servers. Use `"modern"` for best compatibility.
    </Warning>

    Backend servers must also be configured to accept the chosen forwarding mode. For `"modern"` forwarding, set `velocity-forwarding-secret` in your Paper server's `config/paper-global.yml`.
  </Step>

  <Step title="Start the proxy">
    Start or restart your Velocity proxy. Geyser will generate its configuration file at:

    ```
    plugins/Geyser-Velocity/config.yml
    ```
  </Step>

  <Step title="Configure Geyser">
    Open `plugins/Geyser-Velocity/config.yml` and review the settings. Adjust as needed:

    - `bedrock.address` — the IP Geyser listens on (default `0.0.0.0`)
    - `bedrock.port` — the UDP port Bedrock clients connect to (default `19132`)

    See the [configuration reference](/configuration/config-reference) for a full description of every option.
  </Step>

  <Step title="Open the Bedrock port on your firewall">
    Bedrock clients connect over **UDP**. Make sure port `19132` (or whichever port you configured) is open on your proxy's firewall and forwarded on your router if applicable.

    <Note>
      Unlike Java Edition (TCP), Bedrock uses UDP. Many firewalls and hosting panels block UDP by default. Ensure both OS-level and panel-level firewall rules allow inbound UDP on the Bedrock port.
    </Note>
  </Step>

  <Step title="Verify the connection">
    Start your proxy and look for a line in the console similar to:

    ```
    [Geyser-Velocity] Started Geyser on 0.0.0.0:19132
    ```

    Bedrock players can now connect using your proxy's IP address and port `19132`.
  </Step>
</Steps>

## Do not install Geyser on backend servers

With this setup, Geyser only needs to run on the proxy. Installing it on backend servers as well will cause conflicts and is not supported.

## Floodgate

[Floodgate](https://geysermc.org/download#floodgate) is an optional companion plugin that lets Bedrock players authenticate without a Java Edition account. For a Velocity network:

- Install Floodgate on the **proxy** (alongside Geyser).
- If backend servers use online mode, install Floodgate on each backend server as well.
